# Steam Buy Order Placer — Инструкция

Скрипт для автоматического выставления buy-ордеров на Steam Market.  
Поддерживает несколько аккаунтов, автоматическую авторизацию через браузер и Excel-файл с ордерами.

---

## Требования

- Python 3.9+
- Google Chrome (актуальная версия)

```bash
pip install requests openpyxl undetected-chromedriver
```

---

## Быстрый старт

### 1. Первый запуск (нет сохранённого профиля)

```bash
python steam_buy_order.py --user-profile main
```

Откроется окно Chrome со страницей входа Steam.  
Войди в аккаунт → вернись в консоль → нажми **Enter**.  
Куки считываются автоматически и сохраняются в `steam_sessions/steam_session_main.json`.

### 2. Повторный запуск

```bash
python steam_buy_order.py --user-profile main
```

Браузер **не открывается** — используются сохранённые куки.

### 3. Переавторизация (если куки протухли)

```bash
python steam_buy_order.py --user-profile main --reauth
```

## Файл с ордерами — `orders.xlsx`

### Структура колонок

| Колонка | Заголовок | Обязательная | Описание |
|---------|-----------|:---:|---------|
| A | Item Name | да | Точное название предмета (`market_hash_name`) |
| B | Price | да | Цена в валюте аккаунта (число, напр. `193` или `1.50`) |
| C | Quantity | нет | Количество штук (по умолчанию: 1) |
| D | AppID | нет | ID игры (по умолчанию: 252490 = Rust) |

> Первая строка — заголовок, скрипт её автоматически пропускает.

### Пример содержимого

```
A                              B        C    D
────────────────────────────── ──────── ──── ──────
Item Name                      Price    Qty  AppID
Welded Metal Sleeping Bag      193      1    252490
Year of the Rabbit AR          127      1    252490
Low Poly Pickaxe               142      1    252490
Soviet Carpet                  5950     1    252490
Salvaged Shelves               85       2    252490
```

> **Важно:** Цена — в валюте твоего Steam аккаунта (KZT, RUB, USD и т.д.).  
> Скрипт определяет валюту автоматически при первой авторизации через браузер.

### Где взять правильное название предмета

Открой страницу предмета на Steam Market — название берётся из URL:
```
steamcommunity.com/market/listings/252490/Welded%20Metal%20Sleeping%20Bag
                                          ↑ это и есть market_hash_name
```
В Excel вставляй уже декодированное (без `%20`): `Welded Metal Sleeping Bag`

## Файл с прокси — `proxies.txt`

> Прокси по умолчанию **отключены** (`USE_PROXIES = False` в скрипте).  
> Чтобы включить — измени на `True` и укажи путь в `DEFAULT_PROXIES_FILE`.

### Формат строк

```
# С авторизацией (рекомендуется):
ip:port:login:password

# Без авторизации:
ip:port
```

### Пример файла `proxies.txt`

```
145.223.57.148:6181:mylogin:mypassword
86.107.58.152:7059:mylogin:mypassword
193.25.167.248:5779:mylogin:mypassword
104.239.86.139:6049:mylogin:mypassword
```

> Скрипт ротирует прокси по кругу: 1-й ордер → 1-й прокси, 2-й → 2-й и т.д.  
> При ошибке `429` (rate limit) автоматически переключается на следующий прокси в списке.

## Несколько аккаунтов

Каждый аккаунт — отдельный профиль:

```bash
# Аккаунт 1
python steam_buy_order.py --user-profile acc1

# Аккаунт 2
python steam_buy_order.py --user-profile acc2

# Посмотреть все профили
python steam_buy_order.py --list-profiles
```

Сессии хранятся в папке `steam_sessions/`:
```
steam_sessions/
  steam_session_main.json
  steam_session_acc1.json
  steam_session_acc2.json
```

## Все параметры

```
--user-profile / -u   Имя профиля аккаунта (по умолч.: default)
--reauth              Принудительная переавторизация через браузер
--list-profiles       Показать все сохранённые профили

--excel               Excel файл с ордерами (по умолч.: orders.xlsx)
--appid               AppID игры (252490=Rust, 730=CS2)
--currency            Переопределить валюту вручную (0 = из профиля)

--delay               Пауза между ордерами в сек (по умолч.: 1.5)
--timeout             Таймаут запроса в сек (по умолч.: 20)
--retries             Попыток на один ордер (по умолч.: 3)
--max-orders          Макс. ордеров за один запуск (0 = все)

--skip-validate       Пропустить сетевую проверку сессии (ускоряет старт)
-v / --verbose        Подробный вывод
```

## Примеры

```bash
# Rust, профиль main, все ордера из orders.xlsx
python steam_buy_order.py --user-profile main

# CS2, другой excel, не более 20 ордеров за раз
python steam_buy_order.py --user-profile main --appid 730 --excel cs2_orders.xlsx --max-orders 20

# Быстрый старт без проверки сессии
python steam_buy_order.py --user-profile main --skip-validate

# Список всех профилей с состоянием токена
python steam_buy_order.py --list-profiles
```

## Коды ошибок Steam

### HTTP ошибки

| Код | Значение | Решение |
|-----|----------|---------|
| 200 | OK | — |
| 403 | Куки недействительны | Запусти `--reauth` |
| 406 | Not Acceptable — Steam отклонил запрос | Профиль создан вручную без браузерных куков. Пересоздай через `--reauth` |
| 429 | Rate limit (слишком много запросов) | Скрипт автоматически делает паузу |

### Steam EResult коды

| Код | Значение | Действие скрипта |
|-----|----------|-----------------|
| 1 | OK — ордер выставлен | — |
| 8 | Неверный параметр (цена/валюта) | Проверь валюту аккаунта и цену |
| 9 | Предмет не найден на маркете | [SKIP] — переходит к следующему |
| 25 | Лимит buy-ордеров исчерпан | [SKIP] — нужно отменить старые ордера |
| 29 | Дубликат (такой ордер уже есть) | [SKIP] — это нормально |
| 42 | Предмет не найден (неверный market_hash_name) | [SKIP] — проверь название |
| 84 | Rate limit: слишком много покупок подряд | Пауза 15с, затем следующий ордер |
| 107 | Недостаточно средств на Steam кошельке | [SKIP] — пополни баланс |

## Известные проблемы

### HTTP 406 — Not Acceptable
**Причина:** Профиль создан вручную (без браузерной авторизации) — в нём нет `webTradeEligibility` и других куков, которые Steam требует для маркета.

**Решение:** Пересоздать профиль через браузер:
```bash
python steam_buy_order.py --user-profile main --reauth
```

### EResult 84 — Rate limit по покупкам
**Причина:** Steam ограничивает количество попыток покупки без подтверждения (Steam Guard). Срабатывает когда на аккаунте много подряд быстрых запросов.

**Решение:** Подождать 5–10 минут и запустить снова. Скрипт автоматически делает паузу 15с между ордерами при этой ошибке.

```bash
# Подожди пару минут, затем:
python steam_buy_order.py --user-profile acc_main
```

> **Совет:** Увеличь задержку между ордерами если EResult 84 появляется часто:
> ```bash
> python steam_buy_order.py --user-profile acc_main --delay 5
> ```

## Как долго живут куки

| Данные | Срок |
|--------|------|
| `steamLoginSecure` (JWT) | ~10–15 дней |
| `sessionid` | До закрытия сессии браузером |

Когда токен истечёт, скрипт сам предложит переавторизацию:
```
[ERR] JWT истёк — нужна переавторизация.
```
Запусти с `--reauth` и войди снова.
